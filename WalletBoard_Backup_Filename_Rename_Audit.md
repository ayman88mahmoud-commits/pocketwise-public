## 1. Executive Summary

**Rename is safe to proceed.**

The restore and import pipeline in WalletBoard is entirely filename-agnostic. Restoring a backup file relies exclusively on JSON content — specifically, Swift `Codable` decoding of `WalletDataSnapshot`, followed by `schemaVersion` validation and field-level integrity checks. The filename passed to `.fileImporter` is captured only for display in the preview sheet (`url.lastPathComponent`) and is never parsed, matched, or validated as a guard condition for restoring data.

Changing the generated export filename from `PocketWiseBackup-<date>.json` to `WalletBoardBackup-<date>.json` affects only the suggested name shown in the system share/export sheet. Existing backup files saved with the old `PocketWiseBackup` prefix will continue to import and restore without any code changes, because the importer accepts any `.json` file and validates only content.

The rename is a one-line change in `DataBackupView.makeBackupFileName()` and a one-line update to the `@State` default. No restore logic, JSON schema, test fixture, or CloudKit code needs to change.

## 4. Restore / Import Dependency Analysis

### Import entry point: `DataBackupView.handleImportResult(_:)` (lines 187–218)

```
.fileImporter(allowedContentTypes: [.json], allowsMultipleSelection: false)
  → url.startAccessingSecurityScopedResource()
  → Data(contentsOf: url)                    ← reads bytes, never checks url.lastPathComponent against a prefix
  → JSONDecoder().decode(WalletDataSnapshot.self, from: data)
  → store.makeBackupValidationReport(for: snapshot)
  → BackupRestorePreview(fileName: url.lastPathComponent, ...)   ← filename used only for UI display label
```

**The filename is never inspected as a guard condition.** `url.lastPathComponent` is only passed into `BackupRestorePreview` to display the filename in the confirmation sheet.

### Restore entry point: `WalletStore.importBackupSnapshotFromJSON(_:)` (lines 2871–2883)

```swift
func importBackupSnapshotFromJSON(_ data: Data) throws {
    let decoder = JSONDecoder()
    decoder.dateDecodingStrategy = .iso8601
    let snapshot = try decoder.decode(WalletDataSnapshot.self, from: data)
    try restoreFromBackupSnapshot(snapshot)
}
```

Receives raw `Data` only — no filename parameter, no path, no prefix check.

### Validation entry point: `WalletStore.validateBackupSnapshot(_:)` (lines 3515–3524)

```swift
guard snapshot.schemaVersion > 0,
      snapshot.schemaVersion <= WalletDataSnapshot.currentSchemaVersion else {
    throw WalletBackupError.unsupportedSchemaVersion(snapshot.schemaVersion)
}
```

Validates: schemaVersion bounds, UUID uniqueness per collection. **No filename dependency.**

### `.fileImporter` filter (line 108)

```swift
allowedContentTypes: [.json]
```

Accepts any `.json` file regardless of name prefix. The system picker may filter by extension only.

### `WalletBackupDocument` (lines 458–477)

```swift
static var readableContentTypes: [UTType] { [.json] }
```

`FileDocument` conformance also relies only on `UTType.json` — no filename matching.

## 5. Risk Assessment

**Classification: Safe now**

| Risk factor | Assessment |
|---|---|
| Restore breaks for old `PocketWiseBackup` files | **No risk** — importer accepts any `.json`, restore is content-only |
| Restore breaks for new `WalletBoardBackup` files | **No risk** — same pipeline, only the suggested export name changes |
| Schema validation affected | **No** — schemaVersion is a JSON field, not filename-derived |
| CloudKit affected | **No** — `WalletICloudSyncService` does not use the manual backup filename |
| Test suite affected | **One test path to verify** — `DemoBackupFixtureTests` calls `importBackupSnapshotFromJSON` with the fixture JSON content, not a filename — it will be unaffected |
| User experience regression | **Minimal** — the only visible change is the pre-filled name in the system export sheet. Users who already saved `PocketWiseBackup-*.json` files can still import them without renaming. |

The only genuine caution is forward-compatibility: if any **external tooling or documentation** references the `PocketWise` prefix as a way to identify backup files, those references should be updated in the same or following commit. No such tooling was found in this repository.
